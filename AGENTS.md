# Presentation Workbench / Codex 规则

默认用中文回复和撰写项目说明；代码标识、命令、API 名称和专有名词可保留英文。

## 本地预览

- 公开版默认工作台地址是 `http://127.0.0.1:5174/`。
- VS Code 已配置自动启动工作台；用户关掉网页后，可用 `Cmd+Shift+B` 重新打开桌面页。
- 不要改回 `5173`，避免和私有工作台串端口。

## 工作模式

- Planning：只做主题拆解、资料整理、页面蓝图和制作计划；不写页面代码，除非用户明确要求进入制作。
- Production：制作或修改 deck 页面；画布固定 `1600x900`，页面内部不响应式重排，只允许整体缩放。
- Review：以截图为准检查页面；不要只读 DOM/CSS。优先处理遮挡、越界、截断、空旷、过密和视觉重心问题。
- Maintenance：做收口、清理、归档、系统规则更新和架构重构；默认做减法。

## 页面制作

- 每批优先推进 3-5 页，避免一次性铺满完整 deck。
- 页面设计服务表达目标，不为复杂而复杂。
- 不默认添加总结栏、底部结论条或装饰性模块。
- 交互只在能降低讲解成本时加入。
- 代码、报告、表格、截图等核心材料必须完整可读，不能被装饰挤压。
- 发现越界、遮挡、文本截断或明显视觉失衡时，不交付。

## 目录职责

- `src/`：通用工作台运行逻辑、桌面页、基础样式和标注工具。
- `decks/`：每个汇报独立目录，保存内容、样式、资产、review 和交付物。
- `scripts/`：启动、review、checkpoint 和校验脚本。
- `tests/`：自动检查和工作台交互测试。
- `prompts/`：给 Codex 或其他编码 AI 使用的任务模板。
- `system/`：跨 deck 长期规则、偏好、失败模式和模板。

## 系统收敛

- 修改前先判断是否真的需要新增文件、组件、layout 或脚本。
- 一次性页面结构留在 deck 内，不提升到 `src/`。
- 只有跨 deck 复用成立，才沉淀到 `src/` 或 `system/`。
- 删除历史文件前先列候选清单；优先移动到归档目录。

## 验证

- 仅修改文档时，运行 `npm run verify:docs`。
- 修改页面渲染相关文件时，运行 `npm run check` 和 `npm run review`。
- 修改工作台桌面页、registry、启动脚本或测试时，运行相关测试；默认至少运行 `npm run test:gallery`。
- 不确定影响范围时，扩大验证范围。

## 内置占位

- `placeholder-blank-01` 到 `placeholder-blank-04` 是占位空白 PPT，用来展示工作台收纳多个汇报的能力。
- 不要把这些 deck 描述成正式模版；它们没有实际演示内容。
- 占位空白 PPT 保留网格背景和固定画布即可；用户理解工作台后可以删除。
